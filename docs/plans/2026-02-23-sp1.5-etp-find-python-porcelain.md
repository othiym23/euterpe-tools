# SP1.5: etp-find Binary and Python Porcelain

## Context

SP1.4 completed the SQLite migration and removed all legacy state code. The
scanning pipeline is stable with `etp-csv` and `etp-tree` operating on DB-backed
data. SP1.5 adds a regex-based file search tool (`etp-find`) and replaces the
`scripts/catalog-nas.py` TOML-based orchestrator with a KDL-configured Python
porcelain suite in a new `etp/` directory.

## Part A: etp-find Binary (Rust)

### 1. Add streaming DAO function

**File**: `etp-lib/src/db/dao.rs`

Add `stream_files()` returning a
`Pin<Box<dyn Stream<Item = Result<FileRecord, sqlx::Error>> + Send + '_>>`. Uses
the same SQL as `list_files()` but calls `.fetch(pool)` instead of
`.fetch_all(pool)`. Implemented via a `MapStream` adapter that transforms raw
query rows into `FileRecord` values.

**Deps** in `etp-lib/Cargo.toml`:

- Add `futures-core = "0.3"` (for `Stream` trait; already a transitive dep of
  sqlx)
- Add `regex = "1"`

Unit test: `stream_files_yields_all_records` — insert test data, consume stream,
assert matches `list_files()` output.

### 2. Add finder module

**New file**: `etp-lib/src/finder.rs`

```rust
pub struct FindMatch {
    pub full_path: String,
    pub size: u64,
    pub ctime: i64,
    pub mtime: i64,
}

pub fn matches_pattern(record: &FileRecord, pattern: &Regex) -> Option<FindMatch>
```

Matches pattern against `format!("{}/{}", record.dir_path, record.filename)`.

**Update** `etp-lib/src/lib.rs` to add `pub mod finder;`.

Unit tests for matching, non-matching, root-level files, Unicode filenames.

### 3. Add find operations to ops.rs

**File**: `etp-lib/src/ops.rs`

- Make `format_size` public (`pub fn format_size`)
- Add `stream_find_matches(pool, scan_id, pattern) -> (usize, u64)` — streams
  from DB cursor, prints matching paths immediately, returns count + total size
- Add `collect_find_matches(pool, scan_id, pattern) -> Vec<FindMatch>` — loads
  all matches for --tree/--csv processing
- Add `write_find_csv(matches, output: &str)` — writes CSV (same
  `path,size,ctime,mtime` format) to a file path or stdout when `output == "-"`
- Add `render_find_tree(matches, root, output: &str)` — renders tree to file or
  stdout when `output == "-"`

### 4. Refactor tree.rs for writer parameter

**File**: `etp-lib/src/tree.rs`

Refactor `render_dir` to write to `&mut dyn Write` instead of `println!`. Update
`render_tree_from_db` to pass `&mut io::stdout()`.

Add `render_tree_from_paths(paths, root, writer) -> (usize, usize)` that builds
the tree data structures from a list of `FindMatch` values and calls the shared
rendering code.

### 5. Create etp-find crate

**New files**:

- `etp-find/Cargo.toml` — deps: etp-lib, clap (derive), tokio, regex
- `etp-find/build.rs` — copy of etp-csv/build.rs (git hash embedding)
- `etp-find/src/main.rs`

**Update** root `Cargo.toml`: add `"etp-find"` to workspace members.

CLI:

```
etp-find <directory> <pattern> [--tree=<file>] [--csv=<file>] [--size]
    [--db <path>] [--exclude <name>...] [--no-scan] [-v]
```

Output logic — three scenarios:

1. **No --tree, no --csv**: Stream matches to stdout (one path per line) using
   `stream_find_matches`. Append `--size` summary if requested.

2. **--tree and/or --csv with file paths**: Collect all matches. Print paths to
   stdout. Write tree/CSV to specified files. Append `--size` summary.

3. **--tree=- and/or --csv=-**: Collect all matches. Serialize stdout output:
   tree first (if `--tree=-`), then CSV (if `--csv=-`). Append `--size` summary.
   Default path listing is suppressed when any output goes to stdout via `-`.

### 8. Create etp dispatcher

**New file**: `etp/etp` (executable)

Git-style dispatcher: `etp <cmd> [args...]` → finds `etp-<cmd>` on `$PATH` or in
the script's own directory, then `os.execv` to replace process. Supports
`etp --help` (lists available subcommands) and `etp --version`.

### 9. Create etp-catalog

**New file**: `etp/etp-catalog` (executable)

Evolved from `scripts/catalog-nas.py`. Key changes:

- **KDL config** via `kdl-py` instead of `tomllib`
- **Calls `etp-tree` and `etp-csv` directly** (found on `$PATH`)
- **`--no-scan` for CSV run**: tree scans first, CSV reuses DB
- **`--du`/`--du-subs` replaces `du -sm`**: mode "used" → `--du`, mode "subs" →
  `--du --du-subs`
- **`--db <shared-path>`** for both binaries to share the same DB file
- **Keeps `df -PH`** calls (cheap syscall, no replacement needed)
- **Slashdash (`/-`) replaces `enabled = false`** for disabling scans

Preserves from catalog-nas.py: `Timer` class, `run_cmd` helper, `--dry-run`,
`--scan`, `--verbose` CLI, `resolve_global` env/interpolation logic.
