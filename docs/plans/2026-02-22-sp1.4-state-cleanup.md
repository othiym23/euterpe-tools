# SP1.4: State Cleanup, du Replacement, and --no-scan

## Context

SP1.3 rewired both binaries to the DB-backed pipeline. The old ScanState code
(state.rs, rkyv, brotli) is dead code. SP1.4 removes it, adds size summary
output to replace external `du` calls, and adds `--no-scan` for faster
back-to-back runs.

No import/migration tool needed â€” state files are caches, not user data. First
run does a full rescan. Python script changes deferred to SP1.5.

## Implementation steps

### 1. Remove state.rs and rkyv/brotli dependencies

- Delete `etp-lib/src/state.rs` (452 lines)
- Remove `pub mod state;` from `etp-lib/src/lib.rs`
- Remove from `etp-lib/Cargo.toml`: `rkyv` and `brotli` dependencies

### 2. Remove old functions from ops.rs

Remove these dead functions:

- `resolve_state_path`
- `load_state`
- `save_state`
- `run_scan` (old ScanState version)
- `write_csv` (old ScanState version)
- `render_tree` (old ScanState version)

Remove `use crate::state::...` imports. Keep: `validate_directory`,
`parse_ignore_patterns`, and all DB-backed functions.

### 3. Remove old scan() from scanner.rs

- Remove `pub fn scan()` and its 8 unit tests
- Remove `use crate::state::{DirEntry, FileEntry, ScanState};`
- `scan_directory()` currently returns `Vec<FileEntry>` â€” change to return
  `Vec<dao::FileInput>` directly since `FileEntry` is going away
- Update `scan_to_db()` to use the new return type

### 4. Remove old write_csv() from csv_writer.rs

- Remove `pub fn write_csv(state: &ScanState, ...)` and its 4 unit tests
- Remove `use crate::state::ScanState;` and related imports
- Add unit tests for `write_csv_from_db` to replace lost coverage (use
  `open_memory()` + DAO calls to set up test data, then assert CSV output)

### 5. Remove old render_tree() from tree.rs

- Remove `pub fn render_tree(state: &ScanState, ...)`
- Remove `use crate::state::ScanState;`
- Remove the ScanStateâ†’HashMap conversion code that was only in the old function

### 6. Update db_output_parity.rs integration test

Replace parity tests (which need the old path) with DB-only tests:

- Keep `scan_to_db_captures_all_directories` as-is
- Replace `csv_output_is_byte_identical` with a DB-only test: scan fixture â†’
  `write_csv_from_db` â†’ assert CSV content is correct
- Replace `csv_output_with_exclude_filters_correctly` similarly

### 7. Remove --state from binary CLIs

Remove the hidden `--state` arg and deprecation warning from both
`etp-csv/src/main.rs` and `etp-tree/src/main.rs`.

### 8. Add `--no-scan` flag to both binaries

Add a hidden plumbing flag `--no-scan` to skip the scanning step and use
existing DB data. For back-to-back runs (catalog-nas.py runs tree then csv on
the same directory), only the first needs to scan.

- Both binaries: if `--no-scan`, skip `run_scan_to_db()`, call
  `dao::latest_scan_id(pool, run_type)` to find the existing scan_id
- Add `dao::latest_scan_id(pool, run_type) -> Result<Option<i64>>` to DAO
- Error with a clear message if `--no-scan` is used but no scan exists

### 9. Add size summary output to etp-tree (`--du`)

Add a `--du` flag to `etp-tree` that appends size summary after the tree output.
Two sub-modes based on what catalog-nas.py needs:

- `--du` alone: print total size of the root path (replaces `du -sm <disk>`)
- `--du --du-subs`: also print size per immediate subdirectory (replaces
  `du -sm <subdir>...`)

Output format (human-readable, not matching du's format since SP1.5 rewrites the
script):

```
Size: 1.23 GiB (root)
  456.78 MiB  subdir-a
  789.01 MiB  subdir-b
```

Use two significant digits with appropriate unit (KiB, MiB, GiB, TiB).

DAO additions needed:

- `subtree_size(pool, scan_id, relative_path_prefix) -> Result<u64>`: sum of all
  file sizes under a given directory prefix (including the directory's own
  size). Empty prefix = total for scan.
- `immediate_subdirectory_sizes(pool, scan_id) -> Result<Vec<(String, u64)>>`:
  sizes grouped by top-level subdirectory name
