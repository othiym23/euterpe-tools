# SP2.1: Metadata Reading with Lofty

## Context

SP1 (filesystem scanning, SQLite database, Python porcelain) is complete. SP2.1
is the first metadata subproject: reading audio metadata from files already
cataloged in the database, using the `lofty` crate. This enables downstream
subprojects (query interface, metadata writing, quality checking).

## Scope

- New `etp-meta` binary crate with `scan` and `read` subcommands
- New `metadata` module in `etp-lib` for lofty interaction and tag normalization
- DB migration adding `metadata`, `cue_sheets`, `blobs`, `embedded_images`
  tables
- CAS filesystem storage for embedded images (BLAKE3 hashing)
- Change `replace_files` from delete+insert to UPSERT to preserve file IDs
- Incremental scan: only re-read files whose mtime changed since last metadata
  scan
- Process files in directory order for sequential I/O on spinning disks

## Implementation Steps

### Step 1: DB migration (`etp-lib/migrations/004_metadata_tables.sql`)

Add `metadata_scanned_at` column to files. Create `blobs`, `metadata`,
`cue_sheets`, and `embedded_images` tables. All foreign keys use ON DELETE
RESTRICT per project ADR.

### Step 2: Change `replace_files` to UPSERT

Current delete+insert destroys file IDs, which would conflict with ON DELETE
RESTRICT from new metadata tables. Switch to UPSERT: INSERT ON CONFLICT UPDATE
for existing files, DELETE only for truly removed files (with metadata cleanup
first).

### Step 3: Add dependencies to `etp-lib`

lofty 0.23, blake3 1, serde_json 1, serde 1.

### Step 4: Metadata reader module (`etp-lib/src/metadata.rs`)

Pure synchronous module. Reads audio files with lofty, normalizes tag names to
`lowercase_snake_case`, combines multi-value tags into JSON arrays, extracts
audio properties and embedded images.

### Step 5: CAS module (`etp-lib/src/cas.rs`)

Filesystem content-addressable storage using BLAKE3. Write blob before DB
reference (safety invariant). Prefix directories for hash distribution.

### Step 6: DAO functions for metadata

Query files needing metadata scan, store/retrieve tags, manage blob ref counts,
handle embedded images and cue sheets.

### Step 7: Operations layer

`run_metadata_scan` orchestrates the full scan pipeline. `read_file_metadata`
provides standalone file reading for the `read` subcommand.

### Step 8–11: Binary crate, workspace, docs

New `etp-meta` crate following etp-find patterns. Update workspace members and
CLAUDE.md.

## Testing Strategy

- Tag normalization: pure unit tests, no audio files needed
- CAS: unit tests with in-memory data
- DAO functions: in-memory SQLite via `open_memory()`
- UPSERT replace_files: verify file IDs are stable across rescans
- Metadata reader: generate tagged audio files with lofty's write API in tests
- CLI integration: trycmd tests with fixture files
